模板 1：角色设定 + 清晰指令（基础款）
适用场景：让模型以特定身份完成单一任务（如写文案、解答问题）
【角色】资深Python开发工程师
【任务】解释什么是AI Agent的“记忆系统”，要求：
1. 用新手能听懂的语言，避免专业术语
2. 分“短期记忆”和“长期记忆”两部分解释
3. 每部分配1个简单例子
【格式】用markdown列表输出


模板 2：Few-shot（少样本）Prompt（进阶款）
适用场景：让模型模仿示例输出（Agent 工具调用、信息抽取必备）
【任务】将用户的自然语言查询转化为“城市-日期-需求”的结构化格式
【示例1】
用户输入：北京明天的天气怎么样？
输出：{"城市":"北京","日期":"2026-03-17","需求":"查询天气"}
【示例2】
用户输入：上海2026年3月20日的机票价格
输出：{"城市":"上海","日期":"2026-03-20","需求":"查询机票价格"}
【现在请处理】
用户输入：广州后天的景点推荐

【输出】{"城市":"广州","日期":"2026-03-19","需求":"查询景点推荐"}

模板 3：思维链（CoT）Prompt（Agent 规划核心）
适用场景：复杂任务拆解（如旅行规划、代码编写），引导模型分步思考
【任务】帮我规划一份2天的杭州旅行计划，预算3000元，侧重文化景点
【要求】先拆解任务，再输出计划，步骤如下：
1. 分析核心需求：预算3000元+2天+杭州+文化景点
2. 确定核心景点：筛选杭州的文化类景点（如西湖、灵隐寺、良渚文化村）
3. 规划每日行程：合理分配时间，控制费用
4. 输出最终计划，格式为「天数-上午-下午-晚上-费用」

模板 4：工具调用 Prompt（Agent 核心能力）
适用场景：引导 Agent 判断 “是否需要调用工具”+“调用哪个工具”（后续 Agent 开发高频用）
【角色】AI Agent决策器
【工具列表】
1. 天气查询工具：输入城市名，返回实时气温、天气状况
2. 计算器工具：输入数学表达式，返回计算结果
3. 知识库工具：输入问题，返回企业文档中的答案
【任务】判断用户问题需要调用哪个工具，输出格式为「工具名称-输入参数」
【示例】
用户输入：北京现在多少度？
输出：天气查询工具-北京
用户输入：100*20+50等于多少？
输出：计算器工具-100*20+50
【现在请处理】
用户输入：公司2025年的营收目标是多少？

模板 5：错误修正 Prompt（Agent 复盘核心）
适用场景：让模型修正错误输出（如 Agent 调用工具失败后，引导其重新思考）
【任务】修正以下错误的工具调用指令
【错误输入】用户问“上海明天的天气”，Agent输出：计算器工具-上海
【要求】
1. 分析错误原因：为什么调用错了工具？
2. 给出正确的工具调用指令
3. 总结避免此类错误的方法



问题类型	优化方法
约束不细化	给模型加 “强约束”（如 “只输出 JSON”“用生活例子”），避免输出偏离预期
结构不清晰	给复杂任务加 “结构化拆解要求”（如 “先定义→再必要性→再流程”）
示例不足	少样本 Prompt 至少给 2 个示例，模型更易理解规律
格式不具体	明确 “标题加粗、例子标注、输出类型（表格 / 列表）”，方便 Agent 解析
